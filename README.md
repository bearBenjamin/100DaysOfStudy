  Я сегодня с вопросом к знатокам. Решая очередную кату Codewars, на возврат единственного уникального числа из массива дубликатов, не смог пройти рандомные тесты. Число элементов на каждый новый тест растет на порядок. Первоначальное решение было таким:
  ```
const filter = numbers.filter((item, index, array) => array.indexOf(item) !== index);
  const numResult = numbers.filter((item) => !filter.includes(item));
  return numResult[0];
```
Упало сразу на 10К.
Второй вариант решения: 
```
const unicNumber = numbers.find(item => numbers.indexOf(item) === numbers.lastIndexOf(item))
return unicNumber;
```
Тоже не взяло порог 10К. И после долго поиска в одной статье было озвученно, что для больших данных лучшим вариантом будет связка reduce, c Set и Map. Пример кода слова не иллюстрировал никак. Там было что то с filter похожим на мой первый вариант. Самому написать не получилось, но нашел примера кода, после разбора которого у меня появился третий вариант:
```
const unicNum = numbers.reduce((acc, n) => acc.set(n, acc.has(n)), new Map);
 const result = [...unicNum].reduce((acc, n) => (n[1] || acc.push(n[0]), acc), []);
 return result[0];
```
Связка reduce и коллекции Map дала отличный результат. Тесты упали на 10М. И собственно вопрос как теперь разделить этот поток данных, что бы запустить асинхроно по чуть-чуть? Или здесь куда то в другую сторону надо думать?
